## 🛡️ 5. Secure Server Actions Execution

All data mutations must follow bulletproof, secure backend patterns:

* **Progressive Enhancement:** Bind actions directly to standard HTML `<form action={serverAction}>` so they function without client-side JavaScript.
* **Security & Validation:** Validate all incoming parameters on the server side using validation engines (like Zod) inside the action execution scope.
* **Optimistic UI updates:** Use React 19's `useOptimistic` hook to update the client state instantly before the server mutation resolves.
* **State Management:** Leverage the native `useActionState` hook to cleanly catch, process, and render server-side validation or processing errors directly into the UI.
